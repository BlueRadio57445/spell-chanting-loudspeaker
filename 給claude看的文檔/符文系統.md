# 符文系統實作紀錄

## 架構：四層

| 層 | 職責 | 核心檔案 |
|----|------|----------|
| 資料層 | 符文定義、Port 型別、註冊表 | `rune_enums.gd`, `rune_port.gd`, `rune_base.gd`, `rune_registry.gd` |
| 圖層 | DAG 結構、拓樸排序、環檢測、連線驗證 | `rune_graph.gd` |
| 執行層 | 施法鏈：拓樸排序 → 逐一播音檔(await) → execute → 傳遞 port data | `rune_executor.gd`, `spell_spawner.gd` |
| UI 層 | 手動 node graph 編輯器（不用 GraphEdit） | `rune_canvas.gd`, `rune_node_widget.gd`, `rune_inventory.gd` |

## 檔案結構

```
scripts/rune_system/
├── rune_enums.gd          ← PortType/RuneCategory 列舉 + PORT_COLORS + can_connect()
├── rune_port.gd           ← port 定義（名稱、型別、必要性、max_out_connections）
├── rune_base.gd           ← 符文基類（Resource），子類覆寫 execute()
├── rune_graph.gd          ← DAG：nodes + edges、Kahn 拓樸排序、環檢測
├── rune_executor.gd       ← Node，await 音檔 → execute → port data 傳遞、施法鎖
├── rune_registry.gd       ← Autoload，硬編碼註冊所有符文，create_instance() 複製
├── spell_spawner.gd       ← 法術生成橋接（目前 print 佔位）
└── runes/
	├── starter_runes.gd   ← 4 個起始符文（Q/W/E/R），各輸出 1 Energy
	└── effect_runes.gd    ← 火球術、治癒術、詛咒

scripts/ui/
├── rune_canvas.gd         ← 編輯器主畫布（連線、重新路由、懸停、邊緣捲動）
├── rune_node_widget.gd    ← 單個符文節點 UI（拖拽、port 按鈕、高亮系統）
└── rune_inventory.gd      ← 背包面板（儲存符文實例，自動垂直排列，拖拽邏輯同畫布）

scenes/ui/
├── rune_canvas.tscn
├── rune_node_widget.tscn
└── rune_inventory.tscn
```

## 連線規則

- **所有 port 預設一對一**（`max_out_connections = 1`），保留一對多擴充性
- **input port 永遠一對一**，新連線自動替換舊連線
- **型別匹配**：同型別才能連（`can_connect()`）
- **環檢測**：`add_edge()` 加邊後 DFS 檢查，有環就拒絕

## UI 互動（已實作）

- **Click-Click 連線**：點 port A → 點 port B 建立連線
- **相容 port 高亮**：連線中，相容 port 放大+白邊，不相容暗淡
- **port 連線狀態色**：未連線 = 半透明空心，已連線 = 實心亮色
- **連線懸停高亮**：滑鼠靠近連線 8px 內，線條加粗+提亮
- **重新路由**：點擊已連線的 input port → 拔起連線 → 接到新 port
- **35px 安全圓圈**：重新路由時原位放開 = 恢復，圈外放開 = 斷線
- **Escape / 右鍵取消**：連線中取消，重新路由時恢復原狀
- **邊緣自動捲動**：連線時，滑鼠靠近畫布邊緣 40px 自動平移
- **中鍵 / 左鍵拖拽空白**：畫布平移（僅在畫布範圍內有效，不影響背包）
- **橡皮筋線**：連線中顯示 port 型別色的貝茲曲線預覽

## 背包系統

- 背包本質上是**有自動排列功能的畫布**，顯示相同的 `RuneNodeWidget`
- 儲存玩家持有的符文**實例**（非 Registry 模板副本），符文在背包或畫布二選一
- **背包→畫布**：拖符文節點到畫布區域放開 → 移到畫布指定位置
- **畫布→背包**：拖符文節點到背包面板放開 → 移回背包自動排列
- **Starter 節點**不可移入背包，拖出畫布範圍會彈回
- 背包與畫布完全獨立：點擊/拖拽背包不觸發畫布平移，反之亦然
- `rune_canvas.rune_inventory` 與 `rune_inventory.canvas` 互相持有引用（由 main.gd 注入）

## 施法流程

```
按 Q/W/E/R → RuneExecutor.trigger_starter(slot)
  → is_casting 鎖檢查
  → graph.get_topological_order(starter_id)
  → 逐一：播音檔 → await finished → rune.execute(inputs) → 傳遞 port data
  → casting_finished signal
```
