# 符文設計細節草案

> 此為設計討論中的草案，部分內容尚未與企劃對齊。

## Port 類型

| Port 類型 | 本質 | 說明 |
|-----------|------|------|
| `Energy` | 資源 | 一條線 = 一顆能量 |
| `HP` | 資源 | 血量成本（來源待定：自動扣血 / 放血符文 / HP型起始符文） |
| `Direction` | List\<Vector2\> | 方向清單，空值時預設玩家滑鼠方向 |
| `Target` | List\<Enemy\> | 目標清單 |
| `Position` | List\<Vector2\> | 位置清單 |
| `Spell` | List\<SpellData\> | 法術清單 |
| `Form` | List\<FormType\> | 型態清單 |

所有非資源型 port 本質上都是 list。每個符文自己決定讀幾個（例如衝刺符文只讀 Direction[0]，其他符文可能全讀）。

`Form` 實作為 Dictionary（非 list），key 為屬性名稱，前修飾符文逐步 merge 進去。

## Form 清單

描述每顆投射物本身的性質，可組合。

### 移動方式

| Form | 說明 |
|------|------|
| 子彈 | 直線飛行（預設） |
| 地面 | 停在地面上不移動 |
| 天罰 | 從天降到指定 Position |
| 追蹤 | 自動飛向 Target |
| 環繞 | 繞玩家旋轉 |

### 形體變化

| Form | 說明 |
|------|------|
| 巨化 | 體積增大、碰撞範圍增大 |
| 龍息 | 錐形持續噴射 |

## Direction 轉換符文

改變 Direction list 的數量與分佈，不屬於 Form。

| 符文 | 說明 |
|------|------|
| 散彈 | 一個方向展開為多個方向（隨機散佈） |
| 扇形化 | 一個方向展開為多個方向（均勻扇形） |
| 偏折 | 往180度反方向 |

## 關鍵澄清

1. **執行鏈必須從起始符文開始。** 起始符文按鍵呼叫 `trigger_starter`，不在執行鏈上的符文不會執行。所有符文都需要某條路徑連回起始符文。

2. **投射物有飛行時間會打空，不能輸出 Target。** 只有必中的符文（非投射物瞬發、追蹤型投射物）才能輸出 Target。

3. **所有 port 都是 list。** Spell 本身就是 List，不存在 SpellList 的區分。

4. **衝突處理：後蓋前，或產生好笑的錯誤。** 不阻止任何組合，不做連線階段的相容性警告。

5. **Direction 空值時預設玩家滑鼠方向。** 未來可能有瞄準符文提供 Direction/Target。
